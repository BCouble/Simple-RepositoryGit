Présentation de git

Tu en à marre d'avoir 36 versions de tes applications ?
Fatiguer de perdre des morceaux de code et de devoir les réécrivent tous les quinze jours ?
Tu aimerais comprendre comment font les développeurs pour créer partager et collaborer sur des projets énormes ?
Comment font-ils pour arrêter un développement en cour et reprendre la ou il en était sans avoir perdu le fil ?

POURQUOI PAS GIT & GITHUB !!!!!
5 arguments pour te convaincre !

1 - Tes développements seront sauvegardés en utilisant GitHub en plus depuis que Bill à racheter tu peux stocker tes projets en privé.

2 - Tu ne te souviens plus des développements que tu as fait il y a quelques mois, git te permet de commenter efficacement chaque étape de ton développement.

3 - Tu peux partitionner ton code avec des branches pour faire des tests, tenter d'ajouter des modules, des fonctions sans tous casser.
Tu gardes ta base applicative fonctionnelle, la branche master.

5 - En utilisant git & GitHub tu pourrais collaborer avec d'autre développeurs git te fournis les outils nécessaires pour comparer et fusionner les ajouts dans le temps.

Alors tu nous rejoins ?
